# CPA Nexus Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task in the current session. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the CPA Nexus dashboard and worker in the empty `cpa-nexus` repository.

**Architecture:** Use Next.js App Router for UI/API, Drizzle ORM with SQLite for local persistence, and a separate node-cron worker process for sync and replenishment jobs. Keep CPA HTTP integration, parsing logic, database access, and UI components in separate modules.

**Tech Stack:** Next.js, TypeScript, shadcn/ui-style components, Tailwind CSS, Drizzle ORM, SQLite, node-cron, Vitest, Docker Compose.
