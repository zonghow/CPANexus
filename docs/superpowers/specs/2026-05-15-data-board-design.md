# 数据看板设计

## 目标

新增一个独立的「数据看板」页面，把 CPA 管理页顶部统计迁移过去，并补充历史趋势图。CPA 管理页只保留 CPA 实例管理列表和添加、编辑、启停操作。

## 页面范围

数据看板提供五个顶部指标：账号数量、可用率、5h剩余、周剩余、代理。移除原先统计里的 CPA 实例数量和替补账号数量。

页面支持查看全量启用 CPA 数据，也支持任意多选 CPA 后查看所选 CPA 的聚合数据。未选择 CPA 时等同于全量。可选择的 CPA 使用启用状态过滤，避免停用 CPA 干扰统计。

## 图表

图表使用 Recharts。新增三张折线图：5h剩余、周剩余、可用账号数量。图表展示当前筛选范围的聚合趋势，不限制选择 CPA 的数量。

## 数据

当前的 `quota_snapshots` 是当前态表，CPA 同步时会按实例删除重建，因此不能直接作为历史趋势来源。新增 `dashboard_metric_snapshots` 表，在 CPA 同步成功刷新配额后按 CPA 写入一条历史指标快照。

每条快照保存：CPA ID、账号数量、可用账号数量、5h平均剩余、周平均剩余、代理数量、采集时间。数据看板 API 根据当前筛选 CPA，把每个时间点上各 CPA 的最近快照聚合成趋势点。

## API

新增 `GET /api/data-board`。查询参数 `cpaInstanceIds=1,2,3` 可选；为空时返回全量启用 CPA 的统计和趋势。接口返回 CPA 选项、当前统计、当前选择、折线图序列。

## 验证

新增聚合逻辑单元测试，覆盖全量、任意多选、当前统计和历史趋势聚合。现有测试套件继续通过。
