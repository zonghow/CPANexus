# 异常账号池设计

## 目标

新增一个独立的「异常账号」页面，用来保存从 CPA 中清理出来的异常账号。清理不再等同于永久删除：账号会先保存到本地异常账号池，再从原 CPA 中移除。后续可以把异常账号池中的账号移动到指定 CPA。

## 页面范围

左侧导航新增「异常账号」，路由为 `/exceptions`。页面展示异常账号池列表，每个认证文件占一行，字段包括邮箱、文件名、添加时间、上次报错信息和来源 CPA。

页面顶部提供两个操作：

- `导出`：下载 CSV，内容为每行一个邮箱。
- `删除全部`：清空异常账号池中的所有本地记录。

每行尾部提供两个操作：

- `移动`：选择目标 CPA，将保存的认证 JSON 上传到目标 CPA。上传成功并同步目标 CPA 后，从异常账号池删除该记录。
- `删除`：只删除异常账号池中的这条本地记录，不触碰任何 CPA。

## 账号清理流程

账号管理页中，批量菜单里的「批量清理异常账号」改为把异常账号 portal 到异常账号池。流程为：

1. 根据当前异常账号行定位本地 `auth_files` 记录。
2. 下载远端认证文件；如果下载失败且本地已有 `raw_json`，使用本地 `raw_json` 作为兜底。
3. 将邮箱、文件名、来源 CPA、上次报错信息、认证 JSON 和添加时间写入异常账号池。
4. 从原 CPA 删除远端认证文件。
5. 删除原本的本地账号记录和对应配额记录。
6. 刷新原 CPA 数据。

账号行操作菜单中，现有「删除」改为普通文字颜色，继续执行当前的直接删除行为。其下新增红色「清理」，用于把单个 session portal 到异常账号池，流程与批量清理一致。

## 数据

新增 `exception_auth_files` 表，独立于 `auth_files`，避免异常池账号参与 CPA 账号同步、配额统计和消息推送。

字段：

- `id`
- `source_cpa_instance_id`
- `source_cpa_instance_name`
- `file_name`
- `email`
- `last_error`
- `raw_json`
- `created_at`
- `updated_at`

`source_cpa_instance_id` 只作为来源信息使用；来源 CPA 被删除时，异常池记录仍保留来源名称。`raw_json` 必须保存完整认证文件内容，保证后续移动到任意 CPA 时不依赖原 CPA。

用 `file_name` 做唯一约束，重复清理同一个文件时更新已有异常池记录，保留最新报错和认证 JSON。

## API

新增 `GET /api/exception-auth-files`，返回异常账号池列表。

新增 `DELETE /api/exception-auth-files`，清空异常账号池。

新增 `DELETE /api/exception-auth-files/[id]`，删除单条异常池记录。

新增 `PATCH /api/exception-auth-files/[id]`，请求体包含 `targetCpaInstanceId`。接口把异常池保存的认证 JSON 上传到目标 CPA，成功后删除异常池记录，并同步目标 CPA。

新增 `POST /api/auth-files/[id]` 动作 `portalException`，用于单行「清理」。

扩展 `POST /api/cpa-instances/[id]/auth-files/batch` 动作 `portalExceptions`，用于「批量清理异常账号」。该动作只接收明确的 `authFileIds`，由前端传入已判定为异常的账号。

## UI 状态

异常账号页面遵循现有仪表盘的布局和按钮风格。移动操作使用现有移动账号对话框风格：选择目标 CPA，确认后显示更新态，成功后刷新数据。

账号管理页行操作菜单高度增加，菜单项顺序为：刷新配额、停用/启用、配置代理、移动到、删除、清理。`删除` 使用普通菜单颜色，`清理` 使用当前删除按钮的红色危险样式。

## 错误处理

清理时如果远端下载失败且本地没有 `raw_json`，不删除原 CPA 账号，并提示认证文件内容不可用。

移动异常账号时，如果目标 CPA 已存在同名文件，返回错误，不删除异常池记录。

批量清理按顺序处理；遇到错误时返回错误并保留尚未成功清理的账号。已经成功 portal 的记录保留在异常账号池中。

## 验证

新增 API 测试覆盖：

- 单个账号清理会保存异常池记录、删除远端文件、删除本地 `auth_files` 和配额。
- 批量清理异常账号不再直接永久删除，而是写入异常账号池。
- 远端下载失败时可用本地 `raw_json` 兜底。
- 没有认证 JSON 时清理失败且不删除原账号。
- 异常账号移动到目标 CPA 后上传认证 JSON，并删除异常池记录。
- 目标 CPA 已有同名文件时移动失败且保留异常池记录。
- 清空、删除单条、导出 CSV 每行只包含邮箱。

前端验证覆盖导航可进入异常账号页面、批量菜单文字和行菜单颜色/新增「清理」入口。最终运行现有测试套件和 lint。
